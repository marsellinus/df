1. Executive Summary

This report documents a comprehensive digital forensics investigation into unauthorized data exfiltration from a MinIO cloud storage instance. A coordinated attack resulted in the extraction of 5 critical business files within a 3-second window, demonstrating sophisticated threat actor behavior and inadequate access control mechanisms.

CRITICAL FINDINGS: The attack successfully extracted highly sensitive files including financial reports, technical architecture documentation, customer databases, API keys, and private certificates. The rapid extraction pattern suggests automated tooling and prior reconnaissance.

3. Investigation Methodology

3.1 Forensic Framework

This investigation employed a multi-layered digital forensics framework consisting of:

  • Log Collection Layer: Flask-based service capturing all MinIO access logs in real-time
  • Analysis Layer: Forensic analyzer implementing 5 anomaly detection algorithms
  • Visualization Layer: Dashboard and reporting system for forensic evidence presentation
  • Storage Layer: SQLite3 database for persistent forensic record keeping

3.2 Anomaly Detection Algorithms

Four primary detection mechanisms were deployed:

1. Bulk Download Detection: Identifies rapid sequential file access patterns (threshold: 5+ files/hour)

2. Sensitive File Access: Flags access to files in the sensitive assets watchlist

3. Multi-IP Detection: Detects access from multiple source IPs within short timeframes (threshold: 3+ IPs/hour)

4. Unusual Hours Access: Identifies access during non-business hours (22:00-06:00)

8. Recommendations

8.1 Immediate Actions (0-24 hours)

1. Credential Revocation: Immediately revoke compromised 'attacker1' credentials and rotate all API keys found in api_keys.txt. Revoke access tokens from all systems.
2. Incident Response: Initiate formal incident response protocol. Notify affected customers of potential PII exposure. Determine if customer_database.csv data has been misused.
3. Network Containment: Block source IP 108.97.44.37 at firewall level. Analyze network logs for lateral movement attempts.
4. Certificate Replacement: Reissue all SSL/TLS certificates. Private keys in private_certificates.pem are compromised and must be invalidated.

8.2 Short-term Actions (1-7 days)

1. Access Control Audit: Review all MinIO bucket policies, IAM configurations, and access control lists. Implement principle of least privilege for all service accounts.
2. Monitoring Enhancement: Deploy advanced threat detection for anomalous access patterns. Implement alerts for bulk file operations and after-hours access.
3. Log Retention: Enable immutable logging and increase retention period to 90+ days. Ensure logs are protected from deletion or modification by attackers.
4. Multi-factor Authentication: Enforce MFA for all system accounts accessing sensitive data. Require MFA for programmatic access via API keys.

8.3 Long-term Actions (1-3 months)

1. Data Classification: Implement formal data classification scheme. Mark highly sensitive files with additional encryption and access restrictions.
2. Defense-in-Depth: Implement encryption at rest for sensitive objects. Add network segmentation to isolate storage systems from public internet.
3. Forensic Capabilities: Deploy forensic monitoring system similar to this project as permanent security infrastructure.
4. Threat Hunting: Conduct retrospective threat hunting to identify if attacker accessed other systems or planted persistence mechanisms.

9. Conclusion

This investigation confirms a sophisticated, targeted data exfiltration attack against the MinIO storage instance. The attack demonstrates several indicators of compromise including credential misuse, pre-planned targeting, and rapid bulk data extraction. The successful extraction of 5 critical business files including financial data, technical documentation, customer records, API keys, and cryptographic material represents a significant security breach with severe business and compliance implications.

The forensic analysis framework deployed during this investigation successfully detected 106 anomalous access events and reconstructed the complete attack timeline with precision. This capability provides a foundation for ongoing threat detection and forensic investigation operations.
